                         HOW THE FEATURES WORK
================================================================================

SCENARIO 1: USER SELECTS JANUARY 2025
───────────────────────────────────────────────────────────────────────────────
User Action:                Dashboard State Change:         Result:
1. Click Month dropdown   → selectedMonth = 1              Month selector updates
2. Select "January"       → Month selector shows January   
3. Click Year dropdown    → selectedYear = 2025            Year selector updates
4. Select "2025"          → Year selector shows 2025       
5. useEffect triggers     → loadMonthlyData() called       API fetches data
6. API returns data       → setMonthlyData(data)          January 2025 expenses shown
                          → setExpenses(data.expenses)    

Expense Table Now Shows: All expenses from January 2025 only

───────────────────────────────────────────────────────────────────────────────

SCENARIO 2: USER CHANGES CURRENCY TO EUR
───────────────────────────────────────────────────────────────────────────────
User Action:                Function Called:                Result:
1. Click Settings gear    → setIsSettingsOpen(true)        Settings modal opens
2. Click EUR currency     → handleCurrencyChange('EUR')    Function called
3. setSelectedCurrency    → setSelectedCurrency('EUR')     UI updates
4. Call updateUserCurrency → authAPI.updateCurrency('EUR') API called
5. API updates user       → Database updated               Currency saved
6. setUser() updates      → User context updated           All amounts now show €

Dashboard Now Shows: All amounts with € symbol instead of $

───────────────────────────────────────────────────────────────────────────────

SCENARIO 3: USER ADDS EXPENSE TO SELECTED MONTH
───────────────────────────────────────────────────────────────────────────────
Setup: User has selected January 2025
1. Click "Add New Expense"
2. ExpenseForm opens
3. Date field defaults to: 2025-01-01 (1st of selected month)
4. User enters: Amount=50, Description=Lunch, Category=Food
5. User clicks Save
6. Expense saved to database with January date
7. loadMonthlyData() refreshes
8. Expense appears in January 2025 table

Database: Expense stored with date 2025-01-01
Display: Shows in January 2025 only
Future: User can switch to February, then back to January - expense still there

                        HOW TO USE THE NEW FEATURES
================================================================================

FOR USERS:
───────────────────────────────────────────────────────────────────────────────

To Change Months:
1. Look for "Select Month & Year" section at top of dashboard
2. Click month dropdown and select different month
3. Click year dropdown and select different year
4. Dashboard updates to show only that month's expenses
5. Add expenses button now defaults to selected month

To Change Currency:
1. Click Settings gear icon in header
2. Settings modal opens
3. Find "Currency" section
4. Click desired currency (USD, EUR, GBP, INR, JPY, CAD, AUD, CHF, CNY, SEK, NZD)
5. All amounts update with new currency symbol
6. Currency is saved to your account

To Add Expense to Past Month:
1. Select desired month and year using dropdowns
2. Click "Add New Expense"
3. Date automatically fills with 1st of selected month (you can change it)
4. Enter amount, description, category
5. Click Save
6. Expense appears in that month's table
7. Can navigate to other months knowing data is safe

================================================================================
                        WHAT HAPPENS BEHIND THE SCENES
================================================================================

When you select a month:
1. Dashboard updates selectedMonth and selectedYear state
2. useEffect detects change and calls loadMonthlyData()
3. getMonthString() formats month as "YYYY-MM"
4. API call made to GET /api/expenses/monthly/YYYY-MM
5. Backend filters expenses for that month only
6. Data returned and displayed in tables

When you change currency:
1. You click CurrencySelector component
2. handleCurrencyChange(currency) called
3. API call made to PUT /api/auth/update-currency
4. Backend updates user.currency and user.currencySymbol
5. Response sent back with new currency
6. Frontend updates user state in AuthContext
7. All components re-render with new currency symbol

When you add expense to selected month:
1. ExpenseForm receives defaultDate prop
2. Date field pre-fills with selected month's 1st day
3. You can keep it or change it
4. Expense saved with whatever date you confirm
5. loadMonthlyData() refreshes
6. Expense appears in correct month's table
